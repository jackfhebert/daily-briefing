## Claude Code Workflow

Claude Code is the primary development tool. The general working pattern:

- Claude Code handles the majority of code generation — services, handlers, tests, config files.
- Manual review is applied to architectural decisions and core logic — data models, API contracts, security-sensitive code.
- The PRD (v1.8) and this document serve as the authoritative reference for Claude Code sessions.
- Each Claude Code session should begin with the relevant PRD sections and any prior session notes as context.

## Data Safety

Personal user data — calendar events, emails, knowledge graph, evening captures — only ever exists in the production GCP project. Staging and dev/test environments use synthetic or anonymized data only.

- No production data is ever copied to staging or dev.
- Each GCP project has its own Firestore instance, Cloud Storage bucket, and service accounts.
- Production service account credentials are never used in staging or dev environments.
